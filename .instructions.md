# Dilling Agentic E-Commerce Platform — Build Instructions

## TL;DR
Build a next-gen e-commerce platform for Dilling (organic merino wool & cotton clothing) with a conversational AI agent as the primary UX and a traditional browsable catalog as a secondary UX. Next.js full-stack monorepo, OpenAI-powered chat, Stripe + PayPal payments, PostgreSQL on Railway, multi-language (i18n) and multi-currency from day one.

## Reference Architecture: bricklink project
The Dilling project replicates the proven patterns from the bricklink codebase (`/Users/ussoenoe/bricklink`):
- **Framework**: Next.js (App Router, standalone output), TypeScript, TailwindCSS v4
- **AI SDK**: Vercel AI SDK v6 (`ai`, `@ai-sdk/react`) with `useChat` hook + `DefaultChatTransport` + `streamText` + `tool()` — adapted for OpenAI instead of Anthropic
- **DB**: Prisma with `@prisma/adapter-pg` + raw `pg` Pool (lazy singleton pattern from `bricklink/src/lib/db.ts`)
- **Search**: Meilisearch (same Docker setup, index config pattern from `bricklink/src/lib/meilisearch.ts`)
- **Payments**: Stripe + PayPal (same lib patterns from `bricklink/src/lib/stripe.ts` and `bricklink/src/lib/paypal.ts`)
- **Auth**: Clerk (same lazy-load pattern from `bricklink/src/app/providers.tsx`)
- **Agent UI**: Chat panel as global floating widget (`bricklink/src/components/agent/chat-panel.tsx` pattern) + Shopper panel pattern
- **Knowledge base**: Static KB array pattern (`bricklink/src/lib/knowledge-base.ts`)
- **Docker**: Compose with Postgres + Meilisearch + App services (same structure as `bricklink/docker-compose.yml`)
- **Makefile**: Same dev/up/down/seed/reset/demo commands
- **Railway**: Same `railway.toml` config with Dockerfile builder
- **Testing**: Vitest (unit/integration/component) + Playwright (e2e) — same config pattern
- **UI components**: Same `src/components/ui/` primitives library pattern
- **Project structure**: Same `src/{app,components,lib,generated}` layout with `@/*` path alias

## Context & Decisions

**Domain:** Dilling — Danish heritage brand (est. 1916). Organic merino wool & cotton clothing/underwear for women, men, children, and babies. ~1200+ products. Categories: activewear (running, skiing, hiking, yoga), underwear, everyday wear, accessories. Prices 29–2500 DKK. Images hosted on Cloudinary.

**Key decisions from alignment:**
- No existing codebase — fully greenfield, but replicating bricklink project structure/patterns
- Product seed data: scrape dilling.com and import
- AI provider: OpenAI (GPT-4o / GPT-4.1) via Vercel AI SDK v6 (`@ai-sdk/openai`)
- Agent scope: product discovery, place orders via chat, FAQ/support, order tracking + interactive features (outfit builder, gift finder, etc.)
- Auth: Clerk (same as bricklink — lazy-loaded, optional)
- Structure: monorepo (frontend + backend in one Next.js app)
- Backend: Next.js API routes only (same as bricklink)
- Search: Meilisearch (same as bricklink — Docker service + typed index config)
- i18n: multi-language from start (Danish primary, English) via next-intl
- Currency: multi-currency from start (DKK primary, EUR, etc.) — free exchange rate API
- DB: PostgreSQL on Railway, Prisma with pg adapter (same as bricklink)
- Deployment: Frontend on Vercel, DB + Meilisearch on Railway
- Payments: Stripe + PayPal (both in MVP, same lib patterns as bricklink)
- Docker: docker-compose with Postgres + Meilisearch + App (same as bricklink)
- Testing: Vitest + Playwright (same as bricklink)
- Design: Match current dk.dilling.com look and feel
- English translations: AI-generated from Danish product data

### Step 1.2 — Database schema (Prisma — generated to `src/generated/prisma`)

- **ProductTranslation**: id, productId, locale (Locale), name, description, careInstructions

- **Category**: id, slug, parentId (self-ref) — relations: translations, products
- **CategoryTranslation**: id, categoryId, locale (Locale), name, description
- **User**: id, clerkId (unique, nullable), email, name, locale, preferredCurrency, sizeProfile (Json — height, weight, fit preference), createdAt
- **Cart**: id, userId (nullable), sessionToken (nullable, for guests), createdAt, updatedAt
- **CartItem**: id, cartId, productId, size, color, quantity, priceAtAdd (Decimal), currency
- **Order**: id, userId (nullable), email, status (OrderStatus), paymentMethod (PaymentMethod), paymentStatus (PaymentStatus), paymentIntentId, totalAmount (Decimal), currency, shippingAddress (Json), locale, createdAt
- **OrderItem**: id, orderId, productId, productName, size, color, quantity, unitPrice (Decimal), currency
- **ChatSession**: id, userId (nullable), sessionToken, context (Json — sizeProfile, preferences), createdAt, updatedAt
- **ChatMessage**: id, chatSessionId, role (String), content (String), toolCalls (Json nullable), createdAt
- **ExchangeRate**: id, baseCurrency, targetCurrency, rate (Decimal), updatedAt

## Phase 2: Product Data & Catalog

### Step 2.1 — Product scraper (*parallel with 2.2*)
- Script: `scripts/scrape-dilling.ts`
- Scrape dk.dilling.com category pages (dame, herre, børn, baby)
- Extract: product name (Danish), SKU (from URL slug e.g. `fg-9926-0611-263`), price (DKK), images (Cloudinary URLs), category, tags (NYHED, ACTIVEWEAR, SVANEMÆRKET, etc.), color variants, available sizes
- Output: JSON seed file → Prisma seed script
- Rate-limit scraping to be respectful of the source

### Step 2.2 — Product API routes + Meilisearch indexing (*parallel with 2.1*)
- **Meilisearch indexes** (same pattern as `bricklink/src/lib/meilisearch.ts`):
  - `products` index: searchableAttributes [name, sku, categoryPath, material], filterableAttributes [gender, material, tags, category, priceAmount, sizes, colors], sortableAttributes [priceAmount, name, createdAt]
  - Synonyms: merinould→"merino wool", uld→wool, bomuld→cotton, dame→women, herre→men, børn→children
- **API routes**:
  - `GET /api/catalog` — paginated list with Meilisearch search + filters, Prisma fallback (same pattern as bricklink agent tools)
  - `GET /api/catalog/item/[slug]` — single product detail
  - `GET /api/catalog/categories` — category tree
- Full-text search via Meilisearch primary, Prisma `contains` fallback (same as bricklink)

## Phase 4: Conversational AI Agent

### Step 4.1 — Agent architecture (Vercel AI SDK v6 — same pattern as bricklink)
- **AI client** (`src/lib/ai.ts`): `createOpenAI()` from `@ai-sdk/openai` (replacing bricklink's Anthropic proxy)
- **Agent API** (`src/app/api/agent/chat/route.ts`): `streamText()` + `tool()` from `ai` package (same pattern as `bricklink/src/app/api/agent/chat/route.ts`)
- System prompt: Dilling brand specialist — knows products, materials (merino wool properties, care instructions), sizing, sustainability story (own dye house in Bredsten, chemical-free). Personality: warm, knowledgeable Nordic style advisor.
- **Core tools/functions:**
  - `searchProducts(query, filters)` — search catalog via Meilisearch
  - `getProductDetails(slug)` — get full product info
  - `addToCart(productSlug, size, color, quantity)` — add item to cart
  - `getCart()` — view current cart
  - `removeFromCart(itemId)` — remove item
  - `getOrderStatus(orderId)` — track order
  - `getCategories()` — list categories
  - `getFAQ(topic)` — answer common questions (shipping, returns, sizing, care) via knowledge base
- **Enhanced tools for interactive experiences:**
  - `buildOutfit(activity, gender, climate?)` — returns a complete outfit bundle (e.g., base layer + leggings + socks + hat for skiing). Renders as an "outfit card" with total price and "Add all to cart" action.
  - `compareProducts(slugs[])` — returns side-by-side comparison card (material, weight, warmth, price, care)
  - `recommendByMood(mood, gender?)` — maps vibe/mood keywords ("cozy", "active", "minimal") to product filters. Returns curated picks.
  - `getSizeRecommendation(gender, bodyInfo?, previousOrders?)` — uses stored measurements + past orders to suggest sizes. Saves preferences to user profile.
  - `findGift(budget, recipientGender, recipientAge?, occasion?)` — curates gift selection within budget, can include gift box (49.99 DKK)
  - `getMaterialInfo(material)` — returns rich educational content about merino wool, cotton, wool/silk, cashmere etc. Includes care instructions.
  - `getSeasonalPicks(season?, activity?)` — date/weather-aware recommendations. Agent proactively uses this on first message.
- Session memory: conversation history stored in ChatSession/ChatMessage tables
- **User preference memory:** store size profile, preferred materials, past purchases, style preferences in User model. Agent loads on session start for returning users.
- Context window management: summarize old messages when approaching token limit

### Step 4.2 — Chat API (mirror bricklink agent pattern)
- `POST /api/agent/chat` — `streamText()` with `tool()` definitions and `zod` input schemas (exact same pattern as `bricklink/src/app/api/agent/chat/route.ts`)
- `POST /api/agent/shopper` — dedicated shopping assistant endpoint (same as bricklink's shopper agent)
- Handles tool calls server-side via Vercel AI SDK tool execution
- Rate limiting per session
- Input validation & sanitization (prevent prompt injection)
- Voice transcription endpoint: `POST /api/agent/voice` — accepts audio blob, transcribes via OpenAI Whisper, feeds to chat pipeline

### Step 4.3 — Chat UI (mirror bricklink chat-panel.tsx + shopper-panel.tsx)
- **ChatPanel component** (`src/components/agent/chat-panel.tsx`): global floating widget using `useChat()` from `@ai-sdk/react` with `DefaultChatTransport` — same pattern as bricklink
- **ShopperPanel component** (`src/components/agent/shopper-panel.tsx`): dedicated shopping assistant panel — same pattern as bricklink
- Both mounted in `providers.tsx` (same as bricklink: `<ChatPanel />` + `<ShopperPanel />`)
- Full-page chat view also at `/[locale]/chat`
- **Message types & rich cards:**
  - Text bubbles (user + assistant)
  - Product recommendation cards: image, name, price, size selector, "Add to Cart" button
  - **Outfit bundle card**: grouped products with outfit image, individual + total price, "Add all to cart"
  - **Comparison card**: side-by-side table (2-3 products)
  - **Quiz/guided flow**: quick-reply button groups for needs assessment ("Who's this for?" → Dame/Herre/Børn → "What activity?" → Hiking/Skiing/Yoga/Everyday → personalized results)
  - **Material education card**: expandable card with material properties, care tips, sustainability info
  - **Gift suggestion card**: curated products + gift box option + budget summary
  - Cart summary inline
- **Quick-action buttons** (shown on empty chat / as suggestions):
  - "Dress me for..." (skiing / hiking / yoga / everyday)
  - "Find a gift"
  - "What's merino wool?"
  - "What's in my cart?"
  - "Track my order"
- **Voice input**: microphone button using Web Speech API (SpeechRecognition) for real-time speech-to-text, with Whisper fallback for unsupported browsers
- Typing indicator, streaming text display
- Chat history persistence (reload continues conversation)
- **Proactive greeting**: on first visit, agent greets with seasonal context ("Welcome! It's ski season — want to see our warmest merino base layers?") using `getSeasonalPicks`

### Step 4.4 — Guided shopping flows (agent behaviors)
These are primarily **system prompt instructions + tool combinations**, not separate infrastructure:

**A. Needs assessment quiz**
- Agent detects broad intent ("I need new clothes") → triggers guided flow
- Quick-reply buttons: Recipient → Activity → Climate/Season → Budget → Results
- Agent calls `searchProducts` with accumulated filters, returns curated grid

**B. "Dress me for..." mode**
- User says "I'm going skiing in Norway" or clicks quick-action
- Agent calls `buildOutfit(activity="skiing", gender, climate="cold")`
- Returns layered outfit: underwear → base layer → mid layer → socks → accessories
- Each piece is selectable/swappable, with "Add all to cart" for the bundle

**C. Gift finder**
- "Help me find a gift for my mom" → agent asks budget, occasion, her style
- Calls `findGift(budget, gender="dame", occasion)` → curated selection + gift box

**D. Size advisor**
- First interaction: "What's your height and preferred fit (slim/regular/relaxed)?"
- Stores in user preferences (or session for guests)
- All subsequent product recs auto-filtered to correct size
- For returning users: "Last time you wore M in activewear — same this time?"

**E. Mood/vibe discovery**
- "I want something cozy" → `recommendByMood("cozy")` → fleece, soft knits, wool/silk
- "Something for an active weekend" → activewear line
- Maps natural language vibes to product attributes

**F. Material educator**
- "What's special about merino wool?" → `getMaterialInfo("merino")` → rich card
- Covers: temperature regulation, moisture-wicking, anti-odor, sustainability (Dilling's dye house), care instructions
- Can be triggered proactively when showing wool products to educate

**G. Proactive seasonal suggestions**
- Agent checks current date + user locale on session start
- Winter → ski/hiking base layers; Spring → activewear; Summer → cotton basics
- Returning users: "You bought merino boxershorts 6 months ago — time to restock?"

### Step 4.5 — FAQ & support knowledge base (mirror bricklink/src/lib/knowledge-base.ts)
- Same `KBArticle` interface pattern: `{ id, title, category, keywords[], content, url }`
- Static `KNOWLEDGE_BASE` array in `src/lib/knowledge-base.ts` with Dilling-specific articles:
  - Shipping policy (free over 499 DKK to pakkeshop)
  - 100-day returns
  - Care instructions (wool at 30°, cotton higher temp)
  - Brand story (since 1916, own dye house in Bredsten, chemical-free)
  - Material properties (merino wool, cotton, wool/silk, cashmere)
  - Sizing guides per category
  - Sustainability / certifications (Svanemærket / Nordic Swan)
- `searchKnowledgeBase(query)` function — keyword matching (same as bricklink)
- Extensible: can add more articles without code changes

## Confirmed Decisions
- **Checkout via chat:** Agent manages cart in chat, then redirects user to `/checkout` page for final payment (PCI compliance + familiar UX)
- **Scraper:** Rate-limited one-time run for seed data; reference Cloudinary image URLs directly (no re-hosting)
- **Agent streaming:** Stream token-by-token for responsive UX, with tool call results batched
- **API keys:** Replicate from bricklink `.env.example` pattern — stub with placeholders, fill later
- **English translations:** AI-generated from Danish product data (via OpenAI during seed)
- **Exchange rates:** Free API (exchangerate-api.com) for live rates
- **Design:** Match current dk.dilling.com look and feel
- **Auth:** Clerk (same as bricklink) — replaces original NextAuth plan
- **Search:** Meilisearch (same as bricklink) — replaces original PostgreSQL full-text plan
